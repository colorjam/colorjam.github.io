To Be a Thinker

Apr 27, 2017

确定了自己想进一步学习的方向是交互设计后，报了线上的手绘班，还交了暑期培训班的订金。两天的练习就消耗完一把辉柏嘉399，等快递同时无所事事想想接下来到底应该做什么。明知道夏令营最看重的是作品集却迟迟没有做好准备，下载了一个学姐的作品集，顺藤摸瓜到她的LOFTER一只汪的世界，看完她的一些日志，颇有感触。在我看来已经是很厉害的人儿了，但也一直在思考和进步着。

来自学姐的一点小总结和我的小思考

  • 从“懒”，自以为是的努力，到学会主动思考，让努力变得有效。

    数字陷阱在我们周围广泛存在，很多人以此来自我安慰、麻醉，甚至我们经常都在有意无意地增强这种影响。

    比如，我看到同学A每天6点起床、12点睡觉，用6个月时间写完了十万字的论文+毕设，在同学中耗时最久产出最大。所以我得出结论他的毕设一定做的特别厉害，我只有两个月的时间，做的肯定比他差。而且不只我这么想，大家都这么想。于是大家都认为A的毕设会是最好的，A自己也认为他花了那么多时间和经历，成果必然是拔尖的。但结果出来，A并不是最好的那一个，甚至与大家的想法差距很大。

    人们经常被各种累积的数据所骇到，进而想当然一个结果。这是很常见的现象，而且我通常就是作为那个A的角色。在很长时间，我都陷入在数字陷阱中无法自拔，比如我会花大量的时间来“学习”，像收集勋章一样去参加各种比赛最后成了赛棍，跑很多地方去听讲座顺便social… 这种在经历上不停累加各种数据的行为，看起来很漂亮，但却支撑不起一个设计师的全部。

    根据二八定律，一件事的的关键往往集中在20%的内容中，在设计中，这20%就是独立深入的思考与逻辑清晰的表达， 这也是我之前最缺乏的东西。一个完整的设计过程通常包括了定义、发现、综合、建构、精化、反思，在之前的过程中，我在从定义到建构之间，反反复复，一直缺少精化和反思。我知道在细节处理上我很弱，但是一直不知道怎么去做，或者说对于复杂问题的处理，我觉得只要付出了足够的时间就够了。当我正视到数字陷阱的虚假表象之后，才正视到设计工作的核心。

    ​

看到这里我清楚的意识到我一直存在的焦虑，便是因为我也存在着”数字陷阱”，一方面被自己的低效率困扰，在一件事情上花了大量的时间却总是做不好，另一方面又不愿意开动脑筋发散思维、深入思考。我自以为是的热爱学习，不过是记好看的笔记，然后强记硬背，而不是真正的融会贯通，掌握以后形成自己的知识体系。

  • 从害怕、逃避，到主动解决问题

    设计的工作是能够发现问题，解决问题，并让解决方案能够真正执行下去。然后进入新一轮的问题发掘中。所以在设计的过程中，问题是不可避免、且永远要面对的，是没有标准答案的。

    而我，或者我身边的一些人，都希望能够拿出一个“没有问题”的解决方案，对于复杂的、超出当前能力范围的问题，就会放弃、崩溃，告诉自己我做不到。这种“非黑即白”的处事方法在设计过程中是非常不可取的。因为我们面对的问题，往往都包含了一些全新的、未知的元素，不会让我们动动手指就能解决，也不会难到无法可解。

    对于这类有挑战的难题，我以前不得不面对的时候，我就是正面刚，用时间使劲刚，但其实我内心还是拒绝的。最后会有一些成果，但基本都是无意义的。

针对这个问题学姐提出了从以下几个方面入手：

  1. 收集尽可能多的信息，将问题定义清楚。

    一个好的问题，通常比一个好的方案更有价值。尤其在面对一个新的问题时，弄清其目的、意义、甚至问题本身是否需要去优化，都是值得去深入挖掘的。而不是拿到问题之后就立刻开始进入到”做”的阶段。

  2. 找寻事物之间的练习，选择特定的角度，深入挖掘联系。

    如果把一个问题的相关因素看做一组点，在一个三围空间中将这些点按照与问题的相关性排布，那么就可以带到一个离散型分布的特定空间。旋转这个三围空间，就可以得到在不同的视野下观察到的图像。在一些特定的视野中，发现这些点的分布具有某种有趣的规律，于是就会以这些视野为关键研究对象，继续寻找更多的相关信息、发掘其背后的原因、推导出结论等等。

    这里对于散点观察选择的视野，对应的就是我们在设计时选择的角度。同一件事物，不同的行事目标决定了要从不同的角度观察，会看到不一样的内容，最终也会带来不一样的结果。

  3. 找到解决问题的思路，不断接近问题与答案。

    这一步才是真正动手去“做”的阶段，在“做”的时候需要不断保持思考的状态，让设计产出的结果和前面的获得的信息能够对应，并且不断去审视和判断之前的研究过程，必要时甚至推倒重来。

    这种解决问题的方式，不是设计师所独有的，而是一种做事的重要素质与能力，但在设计师身上尤其需要强调。

    为什么会如此重要？设计的时候有发散和收敛两个过程，在我们长期的教育和训练中，大部分人是比较擅长收敛的，而且这个过程是很容易被训练的。而发散，一种和创新紧密相关的思维过程，往往难以抓住、稍纵即逝。怎样激发、抓住这些灵感。这是我认为在设计中最有趣、也最耗脑的过程，其中的诀窍近乎玄妙，但有一点，在我对于问题了解的越清晰、相关内容了解越多的时候，灵感也是越多的。

我想，我的作品集便是一次设计的过程，我有意识的明白不能像以往那样展示所做的项目了，从已有的项目延伸出对所做产品的总结，更应该拿出设计师的态度，展示一个从提出问题到解决问题的过程。

  • 从忙乱、无序、拖延，到规划。

    不得不承认的是，在各种设计问题之外，我所面临的最大敌人就是时间——永远做不完想做的东西。而且我发现在工作之后，更可怕的事情是，我已经想不起来在一天之前做了什么。这种状态会让自己陷入一种无意识的忙碌中，发现时间转眼即逝，甚至又进入新一轮的数字陷阱。

    时间管理也是一个很庞大的话题，其实没有找到很好的解决方式，目前的经验是会把每天做的事情都写在evernote中，每天一篇日志，连同当天遇到的问题等等。做完了的事情就打个勾，这样出于强迫症就会忍不住加快速度做事；每周写周报的时候，就会把上一周的日志都拿出来看一遍，大概清楚自己都在忙些什么。

    另外关于习惯的养成，在工作时会尽量不去碰手机，甚至把沉迷的游戏都装在另一台不常用的手机上，很多时候效率低下，是不够专心的结果。

关于时间规划，我也一直摸索着，入了手账坑之后也尝试过很多形式，Bullet Jornal、Chronodex、To Do List……在败家的路上永不停歇，入了很多胶带但是也有点儿偏离了手账的初衷了。本是为了更好的规划和记录生活，但在装饰和拍摄上会消耗大量的时间和精力，和做手账的目标背道而驰了。总结一下目前的手账体系：

  1. 【禾木家-2017彩色周记本】：每日To do List + 小型拼贴
  2. 【KEEP A NOTEBOOK-11-时间轴】：记录每天的日程 + 好句
  3. 【KEEP A NOTEBOOK-02-方眼】：重大事件拼贴
  4. 【KEEP A NOTEBOOK-08-专案】：读书笔记

在接下来的一段时间应该就保持这样的记录风格吧。

  • 从完美主义，到学会取舍、承担

    其实我是一个害怕出错、害怕暴露弱点的人，所以会尝试各种方法来掩藏缺陷。越是在熟悉的环境和团队中，随着彼此间包容和契合度的提高，越能够巧妙得掩盖身上的问题。但就像墨菲定律一样，越是主动回避的东西，越会在某天猝不及防地来打脸，所以曾经逃避的东西都有一天要面对。对于我来说这个问题就是学会取舍。

    断舍离，难处在于放弃的东西并不是可以弃之不顾，而是能够承担相应的后果。所以之前害怕放弃，其实是因为害怕承担后果，但是设计的过程是最优而不是完美，没有舍弃也就不会有前进。

说到取舍，闺蜜们知道我入了手账坑以后礼物都好送了，统统送胶带 (:ι」∠)，但其实好多不是我的风格，强迫症加不想浪费要统统用掉的心理就导致手账的风格有些混乱，衣服也同理。其实很早就看过《断舍离》，但是要真的做到这种境界还是很难的，虽说想要被喜欢的事物包围但又有一种”以后总会用上”的心理。还有一点也是很大的毛病，不知道是不是完美主义，在写代码的时候达到了极致，集中体现在改class上，参考了别人的代码然后看不爽自己的，就会把之前所用的词语统统改掉，导致很多重复工作。以前觉得百变很屌，殊不知是带来效率低下的根本原因。其实根本还是在没有形成自己的一套标准和风格，容易人云亦云。

唉我一直觉得自己是个没什么特长的人，除了好像比身边的人努力之外——但是前面的分析已经指出，单纯的努力是不够的，而且比我聪明的人其实也在更努力啦。虽然事实就是这么难过，但是对于自己的认知确实是很重要的事。

通过这几个月的实习，脑海中对于自己的认知变得越来越清楚。虽然仍会迷茫与困惑，但是对于自己想要什么、自身的特长与缺陷，都在变得清晰。尽管未来路途漫长，但内心笃定一定会有好的结果。

看完学姐这段最后的感想，内心有强烈的共鸣。上次和瘦谈心的时候也如此表明了自己的想法，她说，其实你会的东西已经比别人多多了，要相信自己是最棒的。嗯，我是最棒的！
